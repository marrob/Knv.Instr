NI-DMM .NET Class Library 20.1 Readme

The .NET class library for NI-DMM provides an IVI class-compliant .NET API interface to the underlying NI-DMM instrument driver API.

Note    The NI-DMM .NET class library does not include the required NI-DMM instrument driver. You must install the NI-DMM instrument driver version 20.1 (minimum) for the NI-DMM .NET class library to function. Refer to NI Drivers and Updates on ni.com and enter DMM into the search field to download the latest version of the NI-DMM instrument driver.
Note    The NI-DMM .NET Class Library Help is no longer integrated into the Visual Studio Help Viewer. Select Start»All Programs»National Instruments»NI-DMM»NI-DMM Documentation to view the help.

Installing the NI-DMM .NET Class Library

This section lists the system requirements and installation instructions for the NI-DMM .NET class library.

To use NI-DMM .NET class library support, your computer must have the following:

  • Microsoft operating system:
    • Windows 10/8.11/7 SP12
    • Windows Server 2012 R21
    • Windows Server 2008 R2 SP12
  • Microsoft .NET Framework 4.0 or later

1NI software installs VC2015 Runtime and .NET 4.6.2. Windows 8.1 and Windows Server 2012 R2 require Microsoft updates to support these items. Refer to Microsoft KB2919442 and KB2919355 for more information about how to install these updates.

2NI software is signed with a SHA-256 certificate. Windows 7 SP1, Windows Embedded Standard 7 SP1, and Windows Server 2008 R2 SP1 require Microsoft updates to support SHA-256. Refer to Microsoft KB3033929 for more information about how to install this security update.

Installation Notes

  • You must install the NI-DMM 20.1 (minimum) instrument driver in order to use the .NET API for NI-DMM.
  • Support for Windows 32-bit operating systems may require disabling physical address extension (PAE). To learn how this might affect your system and what actions you might need to take, visit ni.com/info and enter the Info Code PAESupport.
  • If Windows Update is enabled, the installation process might hang if Windows Update interferes with the installation of Microsoft Visual C++ 2015 Run-Time. Visit ni.com/info and enter the Info Code exjq43 for more information and steps to resolve this issue.
  • In 2016, NI dropped support for Windows Vista, Windows XP, Windows Server 2003, and installations of Windows 7 without any service packs. The NI-DMM .NET Class Library will not install or run on an unsupported OS. You cannot deploy or distribute applications that use the NI-DMM .NET Class Library to an unsupported OS. Additionally, after installing the NI-DMM .NET Class Library, you cannot use any installers built on this computer with any version of LabVIEW, LabWindows™/CVI™, NI TestStand™, or Measurement Studio on an unsupported OS.

    For more information about the changes to our OS support for 2016, refer to KB 79UC78LS, Why Does my LabVIEW, LabWindows/CVI, Measurement Studio, or TestStand Built Installer Fail on Windows XP/Vista and Server 2003?.

  • This product supports, and was tested with, the latest Visual Studio service pack and the latest operating system service pack that was available at the time this product was released. National Instruments strongly recommends using NI-DMM with the latest Visual Studio service pack and the latest operating system service pack.
  • If you want to upgrade your operating system from one major version to another, National Instruments recommends first uninstalling all National Instruments software, including application software and drivers. This is typically not necessary when installing an operating system service pack.

Installation Instructions

  1. Download the .zip file that contains the NI-DMM .NET class library.
  2. Unzip the contents of the .zip file to a temporary location.
  3. Launch setup.exe.
  4. Follow the instructions that appear on the screen.

NI-DMM .NET Class Library Incompatibilities with Previous Versions of NI-DMM .NET Class Library

NI-DMM .NET 20.1

  • Exceptions caused by driver errors now contain the error messages from the underlying C driver.
  • Exceptions from the underlying C driver now have IviCDriverException inner exceptions.
  • Warning messages now contain only the warning messages from the underlying C driver.
  • The exception for driver warning code 1073348710 is now raised as a warning rather than an exception.
  • DmmDriverUtility.ClearError has been obsoleted. Use .NET exception handling to obtain error information.

NI-DMM .NET 15.2

  • In order to be compliant with the IVI-2014 specifications, the name of the NI-DMM .NET assembly has been changed from NationalInstruments.ModularInstruments.NIDmm.dll to NationalInstruments.ModularInstruments.NIDmm.Fx40.dll or NationalInstruments.ModularInstruments.NIDmm.Fx45.dll. To ensure you are using the most recent version of the assembly, you must remove the reference to NationalInstruments.ModularInstruments.NIDmm.dll, add a reference to NationalInstruments.ModularInstruments.NIDmm.Fx40.dll or NationalInstruments.ModularInstruments.NIDmm.Fx45.dll, and rebuild your project.
  • DmmMeasurementCompleteDestination.Fromstring has been renamed to DmmMeasurementCompleteDestination.FromString.
  • The SampleTriggerSlope property of the NI-DMM .NET API is not compatible with the 15.1 version of the underlying attribute in the C driver. You cannot use this property in any version of the NI-DMM .NET API with NI-DMM 15.1, because the underlying attribute value is different in NI-DMM 15.1. The value of the NIDMM_ATTR_SAMPLE_TRIGGER_SLOPE attribute expected by the SampleTriggerSlope property in the NI-DMM .NET API is 1150010, but the value of NIDMM_ATTR_SAMPLE_TRIGGER_SLOPE in version 15.1 of the NI-DMM C driver is 1150139. To access the correct sample trigger slope attribute in NI-DMM 15.1, you must use the AdvancedPropertyAccessService with the 15.1 value for NIDMM_ATTR_SAMPLE_TRIGGER_SLOPE.
  • The type for DmmTrigger.Delay has been changed from System.TimeSpan to NationalInstruments.PrecisionTimeSpan to match the type used in the DmmTrigger.Configure method that sets the trigger delay.

Known Issues in NI-DMM .NET Class Library 20.1

The following are known issues in NI-DMM .NET Class Library 20.1. For a list of known issues in the underlying driver, refer to NI-DMM 20.1 Known Issues.

  • An installer created on a 32-bit system using the NI-DMM 20.1 .NET API does not install on a 64-bit system. No error is produced during installation; however, the files for your application will not be present on the target machine after installation.
  • Clicking a link from the NI-DMM .NET Class Library Help to the NI Digital Multimeters Help or the NI-DMM Function Reference Help could cause the Microsoft Help Viewer application to stop working.

    To work around this issue, install the NI-DMM 20.1 (minimum) instrument driver. Refer to NI Drivers and Updates on ni.com and enter DMM into the search field to download the latest version of the NI-DMM instrument driver.

  • The NI-DMM examples may produce a warning - "Warning: Found conflicts between different versions of the same dependent assembly". You can safely ignore this warning. This has been explained in following KnowledgeBase article.
  • When the NI-DMM .NET class library is installed, some assemblies specific to previous versions of NI-DMM could be deleted from the GAC. The affected assemblies are NationalInstruments.ModularInstruments.NIDmm.dll, NationalInstruments.ModularInstruments.Common.dll, and NationalInstruments.ModularInstruments.ModularInstrumentsSystem.dll. Any application which refers to specific versions of these assemblies will not compile once later version of the NI-DMM .NET class library is installed. This has been explained in following KnowledgeBase article.
